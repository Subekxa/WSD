1. What is the Hypertext Transfer Protocol (HTTP)?

    2. The Hypertext Transfer Protocol (HTTP) is an application-layer method of sending hypertext documents including HTML. It was intended to allow web browsers and servers to interact, but it can also be used for other applications.

  2. Why is HTTP called a stateless protocol?

    3. HTTP is known as a stateless protocol because each requirement is implemented individually, with no knowledge of previous requests which implies when the transaction lasts, the communication between both the browser and the server is also lost.

  3. Describe the three phases of establishing a session using client-server protocols such as HTTP?

    4. In client-server protocols, like HTTP, sessions consist of three phases: a. The client establishes a TCP connection (or the appropriate connection if the transport layer is not TCP). b. The client sends its request, and waits for the answer. c. The server processes the request, sending back its answer, providing a status code and appropriate data.

  4. Describe the parameters on a get and post request.

    5. Both the GET and POST methods are used to transmit data from client to the server in the HTTP protocol. The main difference between the two methods is that the GET method appends the request parameter towards the URL string, while the POST method appends the request parameter towards the message body, creating a more stable way of data transfer from client to the server in the HTTP protocol.

  5. What are the HTTP response status code and What do the HTTP status codes, 200, and 404 indicate?

    6. HTTP response status codes show whether or not such a specific HTTP request has completed successfully. The 404 status code is the most well-known among them all, signifying that the requested resource was not found at the given URL, and the server has no understanding how long it took to locate anything.

  6. What are HTTP cookies? Give an example.

    7. An HTTP cookie (web cookie, browser cookie) is a small piece of data that a server sends to the user's web browser. For example, if Path=/docs is set, these paths match: /docs. /docs/Web/ /docs/Web/HTTP.

  7. Why are cookies a security issue?

    8. Cookies aren't damaging because the data their record doesn't change. They can't create viruses or other software to infect computers. Some cyberattacks, on the other hand, can hijack cookies and even get access to your personal data. Their ability to track peoples personal web browsing history poses a risk.

  8. Describe session hijacking and XSS.

    9. A session hijacking attack happens when an attacker takes over your internet session. Attacker can then do anything you could do on the site. In effect, a hijacker fools the website into thinking they are you. Cross-site scripting (commonly known as XSS) is a web security flaw that allows an attacker to manipulate how users interact with a susceptible application.

  9. Describe Cross-site request forgery(CSRF)

    10. Cross-site request forgery (widely known as CSRF) is a web security problem that allows an attacker to mislead users towards doing things they don't like to do. It allows an attacker to completely bypass the same origin policy, which is created to keep websites from engaging with each other.

  10. What are Cross-site HTTP requests (CORS) and what requests use CORS?

    11. CORS (Cross-Origin Resource Sharing) is an HTTP header-based system that allows a server to identify any alternative origins (domain, scheme, or port) through which a browsers can enable resource loading.
